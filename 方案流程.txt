## Claudio 
读懂收听习惯 → 规划声音 → 像 DJ 那样播报”

## 系统架构

自上而下分为四个层级：

1. 第一层：外部上下文

USER：用户品味语料，包含 taste.md、routines.md、playlists.json、mood-rules.md 四个文件。

BRAIN：Claude Code。包含子过程调用、用 Max 订阅无需 API key 以及 claude p | --output json。

MUSIC：NetaseCloudMusicApi（网易云音乐API）。包含音乐检索、直播、歌词、推荐。

VOICE, I/O：Fish、Feishu（飞书）、Weather、UPnP。功能对应为声音、日程、天气、客厅功放。

2. 第二层：本地大脑
这一层包含六个核心 JS 模块及数据库：

ROUTER.JS：意图分流。实现简单指令直连、音乐去 ncm、自然语言走 claude。

CONTEXT.JS：提示词组装。通过 taste + routines + 环境 + 历史 = 生成 system prompt。

CLAUDE.JS：大脑适配器。使用 spawn 子进程，解析 {say, play[], reason, segue}。

SCHEDULER.JS：节律调度。包含 07:00 规划、09:00 早间、小时情绪检查、日历 hook。

TTS.JS：声音管线。Fish Audio -> cache/tts/*.mp3 -> /tts/<hash>.mp3。

STATE.DB：状态·记忆。负责存储 messages、plays、plan、prefs，并支持跨重启持久。

3. 第三层：运行时聚合
中心是一个“CONTEXT WINDOW 组装盒子”，每次触发按以下 6 片粘成 prompt：

① 系统提示词（prompts/dj-persona.md）

② 用户语料（user/*.md）

③ 环境注入（weather, calendar, now）

④ 已检索记忆（state.db, plays）

⑤ 用户输入/工具结果（/api/chat, ncm search）

⑥ 执行轨迹（scheduler, webhook）
下方是 MODEL·前向过程：compute(fragments) -> {say, play[], reason, segue}。引出箭头指示：ncm 解析 queue，tts 合成 say，WS 推 now-playing。

4. 第四层：交互表层

PWA：localhost:8080。描述为 Progressive Web App。包含 Player/Profile/Settings 三视图、单<audio>、WS 流式聊天、sw 缓存壳层、prefetch 10s。

HTTP CONTRACT：PWA <-> server 的 6 条线。具体路径为：POST /api/chat、GET /api/now、GET /api/next、GET /api/taste、GET /api/plan/today、WS /stream。
